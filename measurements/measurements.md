


# All the measurements in *Search Skip Speculate*



Before any measurement, the project's state is committed and tagged. When results then are presented in the paper, they will all provide a tag-id so that it is possible to look in the repository and see the exact state of the project for when the measurment was performed.


The structure of this *measurements*-folder is:


- tag-id:
  - measurement-name
    - all its content
